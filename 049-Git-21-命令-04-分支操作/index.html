语法

  • 比如想把 dev 分支合并到 master 分支，需要

    1. 先切换到 master 分支，然后执行 git merge dev，这个命令会把 dev 分支合并到 master 分支。
    2. 如果有冲突处理冲突。
    3. 冲突解决完成之后依次执行 git add、git commit -m "merge:you info"、git push
    4. 切换到 dev 分支，然后执行 git merge master，这个命令会把刚才合并的代码同步到 dev 分支，并且是解决晚冲突的代码。
    5. 然后就可以接着开发了。

  • 有时候合并操作不会如此顺利。如果你在两个不同的分支中，对同一个文件的同一个部分进行了不同的修改，Git 就没办法合并它们，同时会提示文件冲突。
